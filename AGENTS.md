# Правила для агента

## Работа с токенайзерами в stackTraceSplitter.ts

- **Не модифицируй существующие regexp-ы в массиве `tokenizers`.** Они уже отлажены и покрывают свои кейсы.
- Для поддержки новых форматов стектрейсов **добавляй новые, более специфичные токенайзеры** в массив `tokenizers`.
- Новые токенайзеры должны быть **узкими и точными** — матчить только свой конкретный формат, чтобы не перехватить то, что уже корректно обрабатывается существующими токенайзерами.
- Порядок токенайзеров в массиве имеет значение — первый совпавший токенайзер забирает матч. Размещай новые токенайзеры с учётом приоритета.
